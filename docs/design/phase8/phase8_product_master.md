## 概要
現在の商品一覧画面を「商品マスタ画面」として位置付け直し、公開期間・ステータス管理を追加する。

## 画面構成

### 商品マスタ画面（Amazia Console）
- 商品の登録・価格設定・公開期間・在庫数の設定が可能
- ステータス表示（公開/未公開、販売状況）

### 商品一覧画面（Amazia Market）
- 商品マスタで登録された商品を表示（閲覧のみ）
- 公開期間外の商品は表示しない

## 機能詳細

### 公開期間
| 設定 | 動作 |
|------|------|
| 開始日あり・終了日あり | 期間内のみMarketに表示 |
| 開始日なし | 即時公開 |
| 終了日なし | 恒久的に表示 |

### 商品ステータス
- ステータスはDBに和名とコード値で管理（暫定）
  - `WAITING`（入荷待）
  - `RESERVATION`（予約受付中）
  - `ON_SALE`（販売中）
- 将来的にCSV/DB反映でコード改修なしにステータス追加・編集が可能な設計とする

### 権限（将来対応）
- ログインユーザーへの権限付与で商品マスタ画面の表示制御
- アカウント単位で管理

## DB設計（追加・変更）

### productsテーブル（変更）

| カラム名 | 型 | 説明 | 変更 |
|---------|---|------|------|
| id | BIGINT (PK) | 商品ID | 既存 |
| name | VARCHAR(255) | 商品名 | 既存 |
| description | TEXT | 商品説明 | 既存 |
| status_code | VARCHAR(20) | ステータスコード（FK → product_statuses） | 追加 |
| publish_start | DATETIME | 公開開始日時（NULLは即時公開） | 追加 |
| publish_end | DATETIME | 公開終了日時（NULLは恒久公開） | 追加 |
| created_at | DATETIME | 登録日時 | 既存 |
| updated_at | DATETIME | 更新日時 | 既存 |

### product_statusesテーブル（新規）

| カラム名 | 型 | 説明 |
|---------|---|------|
| code | VARCHAR(20) (PK) | ステータスコード |
| name | VARCHAR(50) | ステータス和名 |
| sort_order | INT | 表示順 |

## TDDテストケース（JUnit / Amazia Core）
- 公開期間外の商品は一覧に含まれないこと
- 公開期間内の商品のみ一覧に返ること
- ステータスコードが不正な値のとき400が返ること

## TDDテストケース（PHPUnit / Amazia Console）
- 公開期間の開始日・終了日を登録・編集できること
- ステータスが選択肢から設定できること
