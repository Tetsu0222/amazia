
# フェーズ13：Amazia Market ログイン・会員登録機能

## ステータス
🔲 未着手

## 範囲
- Amazia Market  
- Amazia Core  
- DB設計

## 機能概要
- Amazia Market にログイン機能・会員登録機能・パスワード再発行機能を追加する  
- HTTPS 化（ALB の DNS 名でアクセス）  
- 郵便番号から住所を自動反映する仕組みを導入  
- Amazia Core に郵便局API連携および住所マスタ登録処理を実装（後にバッチからも利用）

---

## 機能詳細

### 🔐 HTTPS 化
- Amazia Market 全体を HTTPS 化  
- ALB（Application Load Balancer）の DNS 名でアクセス  
- 証明書は ACM を利用（想定）

---

## 🔑 ログイン画面（Amazia Market）
- メールアドレス + パスワードでログイン  
- 認証成功後、ユーザーマイページまたはトップへ遷移  
- ログイン状態はセッションまたは JWT（要検討）

---

## 📝 会員登録画面（Amazia Market）
### 入力項目
- 名前（姓・名）  
- 住所（郵便番号 → 自動補完）  
- 生年月日  
- メールアドレス  
- 決済方法（プルダウン選択）  
  - クレジットカード  
  - 銀行振込  
  - その他（DB or config 管理）

### クレジットカード情報
- **カード番号はハッシュ化して保存**  
  - 生データは保持しない  
  - ハッシュ方式は要検討（SHA-256 + salt など）

### プルダウン選択肢
- DB または config で管理  
- 後述の郵便局API連携で住所マスタを整備

### 郵便番号 → 住所自動反映
- 郵便番号入力時に Ajax で住所検索  
- 住所マスタは Amazia Core が郵便局APIから取得したデータを使用

---

## 🔄 パスワード再発行画面（Amazia Market）
- 本人確認のため **再認証を要求**  
- 再発行後はログイン画面へ遷移  
- パスワードはハッシュ化して保存

---

## ❓ パスワード忘れた場合のフロー
1. メールアドレス入力  
2. メール送信（認証リンク）  
3. 登録情報と一致する場合のみ有効  
4. 認証成功後、パスワード再発行URLを送付  
5. パスワード再発行画面で新パスワードを設定

---

## Amazia Core 側の実装

### 📮 郵便局API連携
- 郵便局が提供する住所データ（CSV）を取得  
- CSV → DB 登録処理を実装  
- 住所マスタとして利用（郵便番号検索用）

### バッチ対応
- 後に定期バッチで住所データ更新を行う  
- Amazia Core の API はバッチからも呼べるように設計

---
