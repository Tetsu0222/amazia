
# フェーズ18：問い合わせ機能

## ステータス
🔲 未着手

## 範囲
- Amazia Console  
- Amazia Market  
- Amazia Core  
- DB設計  
- 通知機能（バッジ表示）

## 機能概要
- Amazia Market からユーザーが問い合わせを登録できるようにする  
- Amazia Console に問い合わせ管理機能を追加  
- Console のヘッダーに「ベルマーク（通知アイコン）」を設置し、問い合わせ件数をバッジ表示  
- 問い合わせはスレッド形式で管理し、ユーザーと管理者が双方向でやり取りできるようにする  
- 将来的に「問い合わせ以外の通知」もベルマークに集約できる設計とする

---

# 機能詳細

---

## 🖥 Amazia Console（管理画面）

### 画面共通項目：ベルマーク（通知アイコン）
- Console 全画面のヘッダーにベルマークを表示  
- 未対応の問い合わせ件数をバッジ表示（例：🔔 3）  
- ベルマークをクリックすると問い合わせ一覧を表示  
- 今後、問い合わせ以外（例：ワークフロー承認待ち、在庫異常、売上不一致など）も表示できるよう拡張性を確保  
- 問い合わせ一覧では「件名のみ」を表示  
  - 件名クリック → 問い合わせ本文（スレッド画面）へ遷移

---

### 問い合わせ一覧画面
- 以下を一覧表示  
  - 件名  
  - ユーザー名  
  - ステータス（未対応 / 対応中 / 完了）  
  - 最終更新日時  
- フィルタ  
  - ステータス  
  - 日付  
  - ユーザー名  
- ソート  
  - 新着順 / 古い順

---

### 問い合わせ本文（スレッド形式）
- メッセージを時系列で表示  
- 管理者は返信可能  
- 添付ファイル対応は将来的に検討  
- ステータス変更（未対応 → 対応中 → 完了）  
- 管理者コメント欄（任意）

---

## 🛒 Amazia Market（ユーザー側）

### 問い合わせ画面
- マイページ内に「問い合わせ」メニューを追加  
- 問い合わせ一覧  
  - 件名  
  - ステータス  
  - 最終更新日時  
- 問い合わせ本文（スレッド形式）  
  - ユーザーがメッセージを投稿可能  
  - 管理者からの返信も表示  
- 新規問い合わせ登録  
  - 件名  
  - 本文  
  - 商品ID（任意）  
  - 添付ファイル（将来的に検討）

# DB設計（追加）

### inquiries テーブル（新規：問い合わせ）

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | BIGINT | PK |
| user_id | BIGINT | 問い合わせユーザー |
| subject | VARCHAR(255) | 件名 |
| status | VARCHAR(50) | 未対応 / 対応中 / 完了 |
| created_at | DATETIME | 作成日時 |
| updated_at | DATETIME | 更新日時 |

---

### inquiry_messages テーブル（新規：スレッドメッセージ）

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | BIGINT | PK |
| inquiry_id | BIGINT | 親問い合わせID |
| sender_type | VARCHAR(20) | user / admin |
| sender_id | BIGINT | 送信者ID |
| message | TEXT | 本文 |
| created_at | DATETIME | 作成日時 |

---

# 技術検討事項
- ベルマークのリアルタイム更新（Polling / WebSocket / SSE）  
- 通知の種類拡張（問い合わせ以外も扱う設計）  
- スレッド形式の UI（吹き出し / タイムライン）  
- 添付ファイルの扱い（セキュリティ・容量）  
- 問い合わせの優先度設定の要否  
- Market 側のユーザー認証と連携  
- 問い合わせの自動クローズ（一定期間返信なし）
