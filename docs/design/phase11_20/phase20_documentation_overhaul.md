### 「学び」セクションの抽出 ★対応
各 trouble の「再発防止」表を横断して、**運用知見として一段抽象化したページ** を新設する。
`docs/troubles/lessons.md` などに配置。

例：
> #### 学び1: 「正常応答」と「正常動作」は別物
> - SSM の `PingStatus=Online` は配信可能性を保証しない（026）
> - Docker コンテナの `running` 状態は中のアプリの ready を保証しない（018）
> - `EC2 status check 3/3 OK` は SSM Agent の MGS セッションを保証しない（026）
>
> → **実コマンド/実リクエストでヘルスチェックする習慣** をつけた。
> deploy.yml ではカナリアコマンドで実配信を検証してから本コマンドを送る構成にした。

> #### 学び2: 「リカバリ機構を強化するたびに次の隙間が露呈する」
> 022（事前検知導入）→ 025（リカバリ後 Pending 滞留）→ 026（ゾンビOnline）の連鎖。
> **対症療法を重ねるのではなく、検証対象そのものを抽象化** することで隙間を塞いだ
> （PingStatus を信じる → 連続Online検知を信じる → 実配信を直接観測する）。

> #### 学び3: 「自分側を疑い尽くしてから外を疑う」は姿勢として正しいが、並行で疑える項目は並行で確認する
> 026 の AWS SSM 障害時の対応（2026-05-05）では、Undeliverable を観測してから
> AWS 側の障害だと特定するまでに余計な時間を要した。
>
> 反省を踏まえ、トラブル発生時は **最初の5分で以下を並行確認** することをチェックリスト化する：
>
> | 項目 | 確認方法 | 所要時間 |
> |------|---------|---------|
> | AWS Service Health Dashboard | https://health.aws.amazon.com/health/status の該当リージョン・サービス | 1分 |
> | EC2 状態 | コンソールで running + 3/3 OK | 1分 |
> | SSM PingStatus | `aws ssm describe-instance-information` | 1分 |
> | セキュリティグループ Outbound | コンソールで 443/0.0.0.0/0 の有無 | 1分 |
> | 最近のデプロイ・設定変更 | git log / CloudTrail | 2分 |
>
> 「過去の失敗から学んで、次回は効率的に切り分ける」ことそのものが
> このプロジェクトのトラブル対応文化の核。

## 🌟 ポートフォリオとしての見せ方 ★対応

このプロジェクトは技術的には「ただの EC2 サイト」ではなく、
- マルチサービス連携（Console/Market/Core）
- 本格的な認証基盤（JWT + refresh + RBAC）
- ECR + SSM 経由の自動デプロイ + ヘルスチェック
- 26件超の実運用トラブル対応

を含む **「実運用シミュレーション」** として作られている。
しかし採用担当者がトップページや README を5秒見ただけでは、その深さが伝わらない懸念がある。
フェーズ20では「**伝わる導線設計**」を最重要目的として位置付ける。

### README トップに「アピール3本柱」を配置
README の最上部に以下の構成を入れる：

```markdown
# Amazia — フルスタック実運用シミュレーション

## 何ができるか
- マルチシステム連携（Console/Market/Core の3アプリが ECR + RDS で連動）
- 本格的な認証（JWT + refresh token + RBAC）
- 完全自動 CI/CD（テスト→ビルド→ECR push→SSM デプロイ→ヘルスチェック）

## 何を学んだか
[26件超の実運用トラブルと対応 →](docs/troubles/README.md)
特に SSM 周辺で 022→025→026 の連鎖を経て、カナリア方式に到達した経緯。
```

### 設計思想
- 「コードが書ける人」ではなく「**システムを運用できる人**」として認識される導線
- troubles/ は本プロジェクトの **最大の差別化要素**。トップから1クリックで辿れること
- 「壊れた → 直した → 二度と起こさない」のサイクルが見えることを最重視

### チェック観点
- README を5秒見て「ただのECサイトじゃない」と分かるか
- README から3クリック以内で troubles/lessons.md に到達できるか
- 各 trouble カテゴリの冒頭1段落だけ読めば、何を学んだかが分かるか
- 構成図が1枚で「3システム + ECR + SSM + RDS」の関係を示せているか
