## 機能概要
- Console でお知らせを登録・管理できるようにする  
- Market のヘッダーにお知らせを表示し、未読をユーザーに通知  
- お知らせは「重要」「普通」の分類を持ち、公開期間を設定可能  
- Market 側ではアコーディオン形式で表示し、未読が複数ある場合は自動切替  
- お知らせ本文はモーダルで表示し、前後のお知らせに移動できる

# 機能詳細

---

## 🖥 Amazia Console（管理画面）

### お知らせ登録画面
- 入力項目  
  - 件名  
  - 分類（重要 / 普通）  
  - 本文  
  - 投稿者（Console の管理者）  
  - 更新日時（自動）  
  - 公開開始日  
  - 公開終了日  
- 注意事項  
  - **投稿者は Market 側には表示しない**

---

## 🛒 Amazia Market（ユーザー側）

### 画面共通項目（ヘッダー）
- ヘッダーに「お知らせ」エリアを表示  
- アコーディオン形式で展開  
- 表示内容  
  - 件名（本文の先頭を表示してもよい）  
  - 文字数が長い場合は見切れてもOK  
- 未読が複数ある場合  
  - **5秒ごとに自動で切り替わる**  
- クリックすると「お知らせ一覧画面」へ遷移

---

### お知らせ一覧画面
- 表示項目  
  - 件名  
  - 分類（重要 / 普通）  
  - 本文  
  - 更新日時  
  - 公開開始日〜終了日  
- 公開期間外（公開前・公開終了）は表示しない  
- 投稿者は表示しない（Console のみ保持）

#### 未読管理
- 本文を開いたタイミングで未読 → 既読に変更  
  - Ajax で既読テーブルにレコード INSERT  
  - レコードが存在すれば既読  
  - レコードがなければ未読  
  - レコードが論理削除されていれば未読扱い  
- 未読数は  
  - **重要**  
  - **普通**  
 で分けてバッジ表示

#### フィルタリング
- 「重要」「普通」をクリックすると分類で絞り込み

---

### お知らせ本文（モーダル表示）
- 一覧画面からモーダルで本文を表示  
- モーダル内に  
  - 「次のお知らせ」  
  - 「前のお知らせ」  
  のボタンを配置し、連続閲覧を可能にする

# DB設計（追加）

### notices テーブル（新規：お知らせ）

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | BIGINT | PK |
| subject | VARCHAR(255) | 件名 |
| category | VARCHAR(20) | 重要 / 普通 |
| body | TEXT | 本文 |
| author_id | BIGINT | 投稿者（Console 管理者）※Market では非表示 |
| publish_start | DATETIME | 公開開始日 |
| publish_end | DATETIME | 公開終了日 |
| updated_at | DATETIME | 更新日時 |
| created_at | DATETIME | 作成日時 |

---

### notice_reads テーブル（新規：既読管理）

| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | BIGINT | PK |
| notice_id | BIGINT | お知らせID |
| user_id | BIGINT | 閲覧ユーザー |
| deleted_flag | TINYINT | 論理削除（1=未読扱い） |
| created_at | DATETIME | 既読登録日時 |

# 技術検討事項
- ヘッダーの自動切替（5秒）を JS で実装するか、Swiper 等のライブラリを使うか  
- 未読管理のパフォーマンス（ユーザー数が増えた場合）  
- 公開期間のバッチ処理 or リアルタイム判定  
- モーダル内の「前へ / 次へ」遷移の実装方法  
- 重要・普通の分類を将来的に増やす可能性  
- 通知エリアのアクセシビリティ（スクリーンリーダー対応）
