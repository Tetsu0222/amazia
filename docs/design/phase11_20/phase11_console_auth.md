
# フェーズ11：Amazia Console ログイン画面

## 範囲
- Amazia Console
- 認証・認可機能
- DB設計

## 機能概要
- Amazia Console にログイン画面を実装
- HTTPS 化（ALB の DNS 名で対応）
- 社員登録・ロール設定・権限管理の仕組みを構築
- Admin アカウントを初期投入し、全権限を付与

---

## 機能詳細

### 🔐 HTTPS 化
- ALB の DNS 名で HTTPS 対応を行う  
  （証明書設定は ACM を想定）

---

### 👤 ログイン画面

#### ■ ログインの流れ
1. 社員登録  
2. アカウント発行  
3. ログイン

#### ■ パスワード再発行フロー
1. パスワード再発行画面でメールアドレス入力  
2. メール認証  
3. 登録情報と一致確認  
4. 該当メールアドレスへ「パスワード再発行URL」を送信  
5. パスワード再発行画面で新パスワード設定

#### ■ 実装機能
- ログイン機能
- パスワード再発行画面

---

### 📝 社員登録画面

#### ■ 入力項目
- ID  
- メールアドレス  
- 名前  
- パスワード  

#### ■ ロール設定
- 管理者 / 一般 の2種類  
- DBで「コード値 + 和名」で管理  
- パーミッションとリレーションできるように FK を設ける

#### ■ 権限設定（パーミッション）
- 画面単位で ID を割り振る
- ロールとパーミッションを紐づける
- 対象ロールに紐づくパーミッションを持つ画面のみ表示可能

---

### 🛠 Admin アカウント
- DB に初期データとして「全権限付与の管理者アカウント」を INSERT する
- ※これを行わないと誰もログインできないため必須

# TDDテストケース（Amazia Core / JUnit）

## 🔐 ログイン機能
- 正しいメールアドレス＋パスワードでログインできること  
- 不正なパスワードの場合、認証エラーが返ること  
- 存在しないメールアドレスの場合、認証エラーが返ること  
- ログイン成功時に JWT（またはセッション情報）が正しく生成されること  
- アカウントが無効化されている場合、ログインできないこと

---

## 🔁 パスワード再発行
- 登録済みメールアドレスを入力すると、再発行メールが送信されること  
- 未登録メールアドレスの場合、エラーが返ること  
- 再発行URLに含まれるトークンが正しく生成されること  
- トークンが期限切れの場合、再発行できないこと  
- 新しいパスワードが正常に更新されること  
- 更新後、旧パスワードではログインできないこと

---

## 👤 社員登録
- 必須項目（ID / メールアドレス / 名前 / パスワード）が揃っていれば登録できること  
- メールアドレスが重複している場合、登録できないこと  
- パスワードのバリデーション（長さ・形式）が正しく動作すること  
- ロール（管理者 / 一般）が正しく設定されること  
- 登録された社員情報が DB に正しく保存されること

---

## 🛂 ロール・パーミッション
- ロールに紐づくパーミッションが正しく取得できること  
- パーミッションIDが存在しない場合、エラーとなること  
- ロールとパーミッションのリレーションが正しく保存されること  
- 権限のないロールでアクセスした場合、アクセス拒否となること  
- 権限のあるロールでアクセスした場合、対象画面が表示可能となること

---

## 👑 Admin アカウント
- 初期投入された Admin アカウントが DB に存在すること  
- Admin ロールに全パーミッションが紐づいていること  
- Admin アカウントで全画面にアクセスできること

---

# TDDテストケース（Amazia Console / PHPUnit）

## 🔐 ログイン画面
- 正しいメールアドレス＋パスワードでログイン成功し、ダッシュボードに遷移すること  
- 不正なログイン情報の場合、エラーメッセージが表示されること  
- ログイン状態でログイン画面にアクセスするとリダイレクトされること  
- ログアウトするとセッションが破棄されること

---

## 🔁 パスワード再発行画面
- メールアドレス入力フォームが表示されること  
- 正しいメールアドレスを入力すると「再発行メール送信完了」メッセージが表示されること  
- 不正なメールアドレスの場合、エラーメッセージが表示されること  
- 再発行URLからパスワード再設定画面が開けること  
- 新しいパスワードを入力すると更新成功メッセージが表示されること

---

## 👤 社員登録画面
- 必須項目が未入力の場合、バリデーションエラーが表示されること  
- メールアドレス重複時にエラーが表示されること  
- 正常に登録すると一覧画面に遷移すること  
- ロール選択が正しく反映されること  
- 登録後、ログイン可能であること

---

## 🛂 権限による画面表示制御
- 権限のあるロールでログインした場合、対象画面がメニューに表示されること  
- 権限のないロールでログインした場合、対象画面がメニューに表示されないこと  
- 権限のない画面URLへ直接アクセスした場合、403エラー画面が表示されること
