# 機能概要
- Amazia Console にサイドメニューを追加し、各画面へ遷移可能にする  
- 商品マスタから在庫管理機能を分離し、専用の在庫管理機能を構築  
- 商品ごとの価格履歴管理（価格マスタ）を導入  
- 商品一覧（Console / Market）のデータ取得元を必要に応じて変更  

---

# 機能詳細

## 1. Amazia Console

### 1-1. サイドメニュー追加
- Console 内に新しいサイドメニューを追加  
- 商品マスタ、価格マスタ、在庫管理など各画面へ遷移可能にする  

---

## 2. 在庫管理（Console）

### 2-1. 商品マスタ（console）
- 商品そのものの種類を登録するマスタ  
- 金額はマスタに保持するが、一覧画面には直接表示しない  
- `code` は自動採番  
- 商品マスタ削除時：在庫数が 0 でない場合はバリエーションエラー  

---

### 2-2. 価格マスタ
商品ごとの価格履歴を管理するテーブル。

#### ■ 登録仕様
- 商品 `code` 単位で価格を登録  
- **初期登録時の自動設定**
  - 金額：商品マスタの金額を初期値として設定  
  - 開始日：商品マスタの開始日を自動設定  
  - 終了日：商品マスタの公開終了日を自動設定  

#### ■ 変更列（将来の価格変更予約）
- 変更後の金額・開始日を設定可能  
- 開始日を迎えたタイミングで、現行価格を上書きする  
  - 将来的には日次バッチで対応  
  - 上書き後、変更列はクリア  
  - 変更履歴も登録したい（別テーブル or ログ）  

#### ■ バリデーション
- 現行の開始日より後でなければ登録不可  
- 商品公開終了日より後の開始日は登録不可  

---

### 2-3. 在庫管理
商品マスタから個数管理を切り離し、専用機能として構築。

#### ■ 基本仕様
- 商品マスタに `code` を付与し、在庫管理と紐付け  
- 商品マスタ登録時：在庫管理側に自動で 0 個として登録  

#### ■ エクセルアップロード
- 同じ `code` が存在する場合は **インサートではなく個数を加算**  
- エクセルは「入荷」を想定しているため、減算は不可  

#### ■ 編集仕様
- 個数の編集は可能  
- 削除は不可（0 個でも履歴として保持）  
  - 将来的に権限者のみ論理削除を許可する可能性あり  

---

### 2-4. 商品一覧（Console）
- 必要に応じて、データ取得元の TBL / SQL を変更  

---

## 3. Amazia Core
- Console / Market の仕様変更に伴い、必要に応じて TBL / SQL を修正  

---

## 4. Amazia Market

### 4-1. 商品一覧
- Console 側の変更に合わせ、データ取得元の TBL / SQL を必要に応じて変更

# フェーズ10：TDDテストケース一覧

---

# 1. Amazia Console

## 1-1. サイドメニュー
### ● サイドメニュー表示
- サイドメニューが表示されること  
- 商品マスタ / 価格マスタ / 在庫管理 / 商品一覧 へ遷移できること  
- 権限に応じて表示制御が正しく行われること（必要なら）

---

# 2. 商品マスタ（Console）

## 2-1. 商品登録
- 商品登録が正常に行えること  
- `code` が自動採番されること  
- 金額がマスタに保存されるが、一覧画面に表示されないこと  
- 初期在庫が在庫管理側に 0 個で自動作成されること  

## 2-2. 商品削除
- 在庫数が 0 の場合のみ削除できること  
- 在庫数が 1 以上の場合、バリエーションエラーになること  
- 削除後、在庫管理側のデータが残る（履歴保持）こと  

---

# 3. 価格マスタ（Console）

## 3-1. 初期登録
- 商品 `code` 単位で価格が登録できること  
- 初期金額が商品マスタの金額で自動設定されること  
- 開始日が商品マスタの開始日で自動設定されること  
- 終了日が商品マスタの公開終了日で自動設定されること  

## 3-2. 変更列の登録
- 変更後金額・開始日が登録できること  
- 現行開始日より前の日付を設定した場合、バリデーションエラー  
- 商品公開終了日より後の日付を設定した場合、バリデーションエラー  

## 3-3. 変更反映（日次バッチ想定）
- 変更開始日を迎えた場合、現行価格が変更後の値で上書きされること  
- 上書き後、変更列がクリアされること  
- 変更履歴テーブル（またはログ）に記録されること  

---

# 4. 在庫管理（Console）

## 4-1. 初期登録
- 商品マスタ登録時に在庫が 0 個で自動作成されること  
- 商品 `code` と在庫が正しく紐づくこと  

## 4-2. 在庫編集
- 在庫数の編集が可能であること  
- 在庫データは削除できないこと（0 個でも保持される）  

## 4-3. エクセルアップロード（入荷）
- 同じ `code` が存在する場合、個数が加算されること  
- 存在しない `code` の場合、エラーまたはスキップ（仕様に合わせる）  
- 減算が行われないこと  
- アップロード後の在庫数が正しく反映されること  
- 不正フォーマットのエクセルでエラーになること  

---

# 5. 商品一覧（Console）

## 5-1. 表示
- 商品一覧が正しく表示されること  
- 取得元の TBL / SQL が変更された場合でも正しくデータが取得できること  
- 商品マスタの金額が表示されないこと（仕様通り）  

---

# 6. Amazia Core

## 6-1. API / SQL 連携
- Console / Market の要求に応じて正しいデータを返すこと  
- TBL変更後も既存APIが正しく動作すること  
- 価格マスタ・在庫管理の新仕様に対応したデータが返ること  

---

# 7. Amazia Market

## 7-1. 商品一覧
- Console 側の変更後も商品一覧が正しく表示されること  
- 新しい TBL / SQL に対応してデータが取得できること  
- 在庫数や価格が正しく反映されること（必要に応じて）
