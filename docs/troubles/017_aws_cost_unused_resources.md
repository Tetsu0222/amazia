# 017: AWS課金 — 不要リソースの蓄積による想定外課金

## ステータス
✅ 解決済（2026-05-05）

## 発症箇所
AWS アカウント全体（EC2・ECR）

## 症状

無料期間中にもかかわらず $133 のクレジットが徐々に消費されていた。

## 根本原因

| リソース | 問題 | 課金内容 |
|---|---|---|
| EC2 本番機 | t3.small（無料枠対象外） | 約 $0.0208/時 → 月 $15 前後 |
| EC2 停止中2台 | stopped でも EBS 課金発生 | 30GB 無料枠を超過した分 |
| EC2 running 1台 | Amazia 無関係の testSample2 | 無料枠 750時間/月を二分割消費 |
| ECR | タグなしイメージが各リポジトリ29個蓄積 | 無料枠 500MB 超過分 |

## なぜ CI で検知できなかったか

AWS コスト監視の仕組みがなく、手動での確認も行われていなかった。

## 修正内容

1. **ECR 古いイメージ削除**: amazia-core・amazia-console 各 27 個のタグなしイメージを一括削除
2. **ECR ライフサイクルポリシー設定**: タグなしイメージを1日後に自動削除するポリシーを両リポジトリに設定
3. **不要 EC2 インスタンス削除**: 停止中 2台・無関係 running 1台の計 3台を terminate
4. **本番機インスタンスタイプ変更**: t3.small → t3.micro（無料枠内）へ変更・動作確認済み

## 再発防止

| 観点 | 対策 |
|------|------|
| ECR イメージ管理 | ライフサイクルポリシーによりタグなしイメージを1日後に自動削除 |
| EC2 管理 | 不要インスタンスは terminated（停止ではなく削除）する |
| コスト監視 | AWS Cost Explorer で月次確認、または Billing アラートを設定する |
| インスタンスタイプ | 学習・デモ用途では t3.micro（無料枠内）を基本とする |
