システム改善・問題解析・パフォーマンス分析などの記録をまとめる。

## 分類

| 分類 | 説明 |
|------|------|
| 障害分析 | 発生したエラー・障害の原因調査と対応記録 |
| パフォーマンス分析 | レスポンス・クエリ・バッチ処理の計測と改善提案 |
| 要件分析 | 機能追加・改修の背景調査と要件整理 |
| セキュリティ分析 | 脆弱性・認証・認可に関する調査記録 |

## 追加ルール

- ファイル名は `YYYYMMDD_<テーマ>.md` 形式とする。
- 同じテーマで複数回分析する場合は連番を付ける（例: `20260503_trouble_analysis_2.md`）。
- 分析結果の対応が完了した場合でも削除せず、`対応済み` のラベルをファイル内に記載する。
